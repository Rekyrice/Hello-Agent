# Hello-Agent 项目入口（给新会话 Agent）

> 目的：当用户在新对话窗口继续本项目时，让 Agent 快速理解项目背景与工作边界，减少上下文消耗。  
> 注意：本文件面向 Agent，不是面向最终用户文档。

## 1. 项目定位

- 项目名称：`hello_agent`
- 项目目标：在当前工作区持续学习、复现并拓展 Agent 能力
- 使用方式：任务驱动（以用户当下目标为准，不预设固定路线）
- 关键原则：先建立项目级认知，再进入当前任务实现

## 2. 工作边界（强约束）

- 仅关注**整体项目推进**与**当前任务目标**，不要默认聚焦某个旧文件的代码细节。
- 具体分析哪个模块，以用户当前需求为准，不依赖历史任务顺序。
- 所有建议应优先“最小改动、可验证、可复用”。

## 3. 环境与配置约定

- 本项目依赖本地 `env` 配置（如大模型 API、搜索 API 等）。
- 所有密钥、凭据、私有配置：
  - 只放本地（如 `.env`）
  - 不允许上传 GitHub
- 修改配置相关逻辑时，优先保持：
  - `.env` 仅本地使用
  - `.env.example` 只保留占位字段，不含真实密钥

### 3.1 当前环境快照（供新会话快速判断）

- 本地 Python 环境：`conda` 环境 `hello-agent`（用户在该环境内运行脚本）
- 当前未发现统一依赖清单文件（如 `requirements.txt`、`pyproject.toml`）
- 按现有脚本推断的主要依赖：
  - 标准库：`os` `re` `typing` `random`
  - 第三方：`requests` `python-dotenv` `openai` `tavily-python`
- 当前可见示例脚本（非完整清单）：
  - `FirstAgentTest.py`
  - `ELIZA.py`

### 3.2 环境操作建议（避免踩坑）

- 新会话若要运行代码，先确认是否已激活 `hello-agent` 环境。
- 若出现 `ModuleNotFoundError`，优先按“缺什么装什么”补依赖，不盲目全量重装。
- 如项目继续扩展，建议新增一份最小依赖清单（如 `requirements.txt`），但仍不提交任何密钥。

## 4. 新会话快速融入步骤（建议流程）

1. 先确认用户“当前要解决什么问题、预期结果是什么”。
2. 再看项目当前状态（结构、最近改动、文档），不要先钻代码细节。
3. 若任务涉及运行：
   - 先确认本地环境与依赖已就绪
   - 再执行最小验证命令
4. 若任务涉及改动：
   - 小步实现
   - 本地验证
   - 根据用户意愿再提交/推送

## 5. 推荐优先参考（按需）

- 当前任务直接相关的脚本与文档
- `docs/` 下的流程/时序文档（用于快速理解历史实现思路）
- 本文件 `AGENT_ENTRY.md`（作为新会话第一参考）

## 6. 与用户协作偏好

- 用户偏好中文交流（包括解释与注释，除非用户明确要求其他语言）。
- 用户可能会频繁开新对话以节省上下文；新 Agent 应先基于本文件建立全局认知，再进入具体任务。

## 7. 交付标准（每次任务）

- 明确：本次任务目标与完成标准
- 明确：是否需要代码改动（不是每次都需要）
- 明确：是否涉及本地配置（严禁泄露/提交敏感信息）
- 明确：下一步建议（运行验证、继续扩展、或收尾）
